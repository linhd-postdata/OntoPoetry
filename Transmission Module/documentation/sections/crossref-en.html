Bibliographic Sourcec back to ToC or Class ToC

IRI: http://postdata.linhd.uned.es/ontology/postdata-transmission#BibliographicSource

A Bibliographic Source is a conceptual entity representing bibliographic products that are generated following a publication process. Instances of pdt:BibliographicSource represent the ideal physical properties of all the exemplars. Despite Bibliographic Sources are not physical objects per se (are not accessible), they encapsulate design properties of the physical carriers (layout, pages, desired material, etc.) and hence they can be "physically" described. An object of type pdt:BibliographicSource embodies one or more textual editions (instances of pdc:Redaction). They mostly represent books that may gather several redactions or poems. In order to represent all the physical copies of the ideal manifestation, we associate this kind of source with instances of pdt:Item. pdt:BibliographicSource is then considered an E73 Informatio Object, which is subclass of E90 Symbolic Object which represents conceptual objects subjected to rights and ownership. Instances of pdt:Item are aligned to E84 Information Carrier.

Source c, e73 information object c

is in range of

Carrier Productionc back to ToC or Class ToC

IRI: http://postdata.linhd.uned.es/ontology/postdata-transmission#CarrierProduction

Instances of pdt:CarrierProduction describe the activity performed by some agent when giving rise to a physical carrier for a textual edition of a poem. A Carrier Production has a set of pdc:AgentRole structures associated indicating the actors and the roles they performed during the participation in the event. The Carrier Production is then carried by taking a textual edition as input (an instance of pdc:Redaction) and generates an Information Carrier (an instance of pdt:InformationCarrier) As a Carrier Production is an activity with input and output objects and participants, it is a subclass of E7 Activity. In addition, being it a type of event involving the creation of a carrier, it is also subclass of E63 Beginning of Existence.
has super-classes
activity c, e63 beginning of existence c
is in domain of
produced op, used redaction op
is in range of
was produced by op, was redaction used by op

Collectionc back to ToC or Class ToC

IRI: http://postdata.linhd.uned.es/ontology/postdata-transmission#OWLClassImpl_8fa2b552_0357_457f_bab1_4d39a073e5a5

A pdt:Collection is an aggregation of instances of pdt:InformationCarrier that are curated and preserved by some pdc:Agent. A Collection can be a set of physical manuscripts or exemplars (i.e. a corpus, a personal collection of books). For example, "Gongocorpus" is a pdt:Collection that collects physical electronic editions of Gongora's Poems. Being a collection a physical aggregation, it has been aligned as a subclass of CIDOC CRM E78 Collection

IRI: http://postdata.linhd.uned.es/ontology/postdata-transmission#Condition

Reproductionc back to ToC or Class ToC

IRI: http://postdata.linhd.uned.es/ontology/postdata-transmission#Reproduction

A Reproduction is an activity in which some actor performs a copy of a physical edition to produce a facsimile. A reproduction of a facsimile is also another facsimile. Instances of pdt:Reproduction have agent-role structures associated (pdc:AgentRole) to indicate the type of role performed by an actor during the reproduction. Reproductions are associated to the carrier being reproduced (of type pdt:InformationCarrier) and generate the copy (instance of pdt:Facsimile) following a specific technique. pdt:Reproduction is aligned to E7 Activity and to E63 Beginning of Existence to indicate that it has input/output objects and it gices rise to a new object.
has super-classes
activity c, e63 beginning of existence c
is in domain of
generated op, reproduction type op, used carrier op
is in range of
was carried used by op, was generated by op
